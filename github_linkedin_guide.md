## Part 1: GitHub Setup & Deployment

### Step 1: Create GitHub Account (if you don't have one)

1. Go to **https://github.com**
2. Click **Sign Up**
3. Choose a professional username (e.g., `abongile-tyenjana` or `atyenjana`)
4. Verify your email address
5. Choose the FREE plan

---

### Step 2: Create Your Portfolio Repository

1. **Log in to GitHub**
2. Click the **"+"** icon (top right) → **New repository**
3. **Fill in repository details**:
   ```
   Repository name: portfolio
   Description: Business Intelligence Portfolio - Power BI Dashboards & Analytics
   ✅ Public (must be public for GitHub Pages)
   ✅ Add a README file
   ```
4. Click **Create repository**

---

### Step 3: Upload Your Portfolio Files

**Option A: Via Web Interface (Easiest)**

1. In your new repository, click **Add file** → **Upload files**
2. **Drag and drop these files**:
   - `portfolio.html`
   - `dashboard_template_guide.md`
   - `implementation_steps.md`
   - `quick_reference.md`
   - `README.md` (the main one from your package)
3. Add commit message: "Initial portfolio upload"
4. Click **Commit changes**

**Option B: Via GitHub Desktop (Recommended for ongoing updates)**

1. Download **GitHub Desktop**: https://desktop.github.com
2. Install and sign in with your GitHub account
3. Click **Clone Repository** → Select your portfolio repo
4. Choose a folder on your computer
5. Copy all your files into this folder
6. In GitHub Desktop:
   - You'll see all files listed as changes
   - Add commit message: "Initial portfolio upload"
   - Click **Commit to main**
   - Click **Push origin**

---

### Step 4: Enable GitHub Pages

1. In your repository, go to **Settings** (top menu)
2. Scroll down to **Pages** (left sidebar under "Code and automation")
3. Under **Source**, select:
   - Branch: **main**
   - Folder: **/ (root)**
4. Click **Save**
5. Wait 2-3 minutes for deployment
6. Your site will be live at: `https://yourusername.github.io/portfolio/portfolio.html`

**🎯 Pro Tip**: Rename `portfolio.html` to `index.html` so your URL is cleaner:
- `https://yourusername.github.io/portfolio/` (instead of `/portfolio.html`)

## Part 2: LinkedIn Integration

### Step 1: Update Your LinkedIn Profile

**1. Add to Your Headline**
```
Business Intelligence Analyst | Power BI Specialist | Data Storyteller
```

**2. Update Your About/Summary Section**

Include a paragraph like this:
```
I specialize in transforming complex data into actionable insights through 
interactive Power BI dashboards. My recent work includes a comprehensive 
sales analytics solution featuring 26+ advanced DAX measures, time intelligence, 
and professional visualizations that drive business decisions.

🔗 View my portfolio: https://yourusername.github.io/portfolio/
💻 GitHub: https://github.com/yourusername
```

**3. Add to Featured Section**

1. Click **Add profile section** → **Featured**
2. Click **+** → **Add link**
3. Enter:
   - **Title**: "Business Intelligence Portfolio"
   - **URL**: Your GitHub Pages URL
   - **Description**: "Interactive portfolio showcasing Power BI dashboards with advanced analytics"
4. Click **Save**

**4. Update Skills Section**

Add these skills (if not already there):
- Power BI
- DAX
- Data Analysis
- Business Intelligence
- Data Visualization
- SQL
- Power Query
- Data Modeling
- KPI Development
- Financial Analysis

Ask connections to endorse these skills!

### Step 3: Create a LinkedIn Article (Optional but Powerful)

Write a longer-form article about your project:

**Title Ideas:**
- "How I Built a Professional Power BI Dashboard for My Portfolio"
- "5 DAX Techniques That Transformed My Analytics Game"
- "From Data to Insights: Building a Sales Analytics Dashboard"

**Article Structure:**
1. **Introduction** - Why you built this project
2. **The Challenge** - What problem does it solve?
3. **The Approach** - Your methodology
4. **Key Features** - Highlight 3-5 standout features
5. **Lessons Learned** - What you discovered
6. **The Results** - Impact/insights
7. **Call to Action** - Link to portfolio

**Publish on LinkedIn:**
1. Click **Write article** (on your homepage)
2. Write your article
3. Add images/screenshots
4. Click **Publish**
5. Share it to your network

## GitHub Pages Troubleshooting

### Issue: Site not loading
**Solution:**
- Wait 5-10 minutes after enabling Pages
- Check Settings → Pages for deployment status
- Ensure repository is Public
- Clear browser cache

### Issue: 404 Error
**Solution:**
- Verify file is named `index.html` (or use full URL with `portfolio.html`)
- Check file is in root directory
- Ensure GitHub Pages is enabled

### Issue: Styling not working
**Solution:**
- All CSS is embedded in the HTML file
- No external dependencies needed
- Should work immediately

## LinkedIn Best Practices

### DO:
✅ Use hashtags (5-10 relevant ones)
✅ Tag companies/people when relevant
✅ Add high-quality images/screenshots
✅ Respond to comments quickly
✅ Post during peak hours
✅ Share authentic insights
✅ Engage with others' content

### DON'T:
❌ Post without visuals
❌ Use too many hashtags (more than 15)
❌ Only self-promote (80/20 rule: 80% value, 20% promotion)
❌ Ignore comments on your posts
❌ Post and disappear
❌ Use generic stock photos
